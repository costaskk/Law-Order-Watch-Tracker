# Catalog, artwork, cast, and ratings maintenance

## Canonical data

`law_order_tracker_app/data/episodes.js` remains the canonical guide used by Python maintenance and serverless matching.

The browser uses optimized outputs generated by `wolf_build_web_data.py`:

```text
episodes.json
wolf_artwork_base.js
wolf_episode_artwork.json
wolf_cast_index.json
```

Do not hand-edit the optimized JSON files. Rebuild them from the canonical sources.

## Rebuild browser data

```powershell
python wolf_build_web_data.py
```

This also enriches locally available TMDB episode ratings, summaries, runtimes, titles, and dates from `tmdb_cache.json`.

## Catalog update

```powershell
python wolf_universe_catalog_update.py
```

Review its report before committing. The unrelated 1992 `CIA` title must not be reintroduced; the project uses `CIA (2026)`.

## Cast and portraits

```powershell
python wolf_fetch_episode_cast.py
python wolf_build_web_data.py
```

The final cast index is compact and lazy-loaded by the browser.

## Episode artwork

Use the catalog/artwork updater or:

```powershell
python wolf_force_episode_artwork.py
python wolf_build_web_data.py
```

Episode stills load asynchronously. Missing stills fall back to season or show artwork.

## Ratings

Set optional provider keys in your local environment:

```powershell
$env:OMDB_API_KEY="..."
$env:TRAKT_CLIENT_ID="..."
```

Then run:

```powershell
python wolf_fetch_episode_ratings.py --write
```

Available providers are displayed independently. Rotten Tomatoes and Metacritic values depend on OMDb availability; many individual TV episodes do not have those ratings.

## Validate after every data update

```powershell
npm test
```

## Release archive

```powershell
npm run release
```

The archive deliberately excludes secrets, local tokens, Git history, caches, workbooks, backup files, debug exports, and historic patch scripts.
